¿Que es NoSql?

  • En uso e implementacion no usan SQL.para realizar Consultas
  • No hay soporte total para ACID
  • ACID (Availability, Consistency, Integrity,Durability)
  • current-visible
  • No existen esquemas fijos de modelos

NoSQL no es un DBMS

"Es un concepto para diferenciarse con los RDBMS. Son bases de datos documentales,clave/Valor, orientadas a grafos, es decir todo se trabaja como un documento estandar de texto."

Base

Basically Available, Soft State, Eventually Consistent

* Se garantiza una respuesta a todas las peticiones sean fallidas o satisfactorias.
* El sistema puede cambiar con el tiempo apenas de no tener entradas.
* Consistencia Eventual : La Db sera consistencial siempre en el tiempo, apesar de inconsistencias presentadas.
